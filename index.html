// MODULE 01 — COVERAGE

Coverage in
SystemVerilog

Coverage is the metric that tells you how much of your design intent has been exercised by your testbench.

// INSTRUCTOR NOTE
Think of coverage as a GPS tracker for your tests. You write stimulus, but how do you know you've hit all the important scenarios? Coverage gives you that answer. Without it, you're flying blind — your simulation might pass but miss critical corner cases that only show up in silicon.
📐
Code Coverage
Structural coverage automatically measured by the simulator. No user code needed — the tool instruments RTL.
// Types: line → every line executed? toggle → 0→1, 1→0 on signals branch → if/else paths taken expr → boolean sub-exprs fsm → state transitions cond → condition coverage
🎯
Functional Coverage
User-defined coverage written in SystemVerilog. Captures design intent — not structure. The most important kind.
covergroup cg; coverpoint addr; cross addr, rw; endgroup // Measures INTENT ✓
🔍
Assertion Coverage
Tracks how many times a cover property or cover sequence was hit. Complementary to functional coverage.
cover property ( @(posedge clk) req ##1 ack ); // count hits ✓
// MODULE 02 — FUNCTIONAL COVERAGE DEEP DIVE

covergroup, coverpoint,
bins & cross

// LESSON 01
COVERGROUP
Basic covergroup
A covergroup is a user-defined type that groups related coverage points. Instantiate it like a class.
SV
covergroup addr_cg @(posedge clk);
  // coverpoint samples 'addr' on posedge clk
  cp_addr : coverpoint addr {
    bins low = {[0:63]};
    bins mid = {[64:191]};
    bins high = {[192:255]};
  }
endgroup

// Instantiate
addr_cg cg_inst = new();
// Call manually if no clk event
cg_inst.sample();
covergroup — defines the coverage container type. Must be instantiated with new()
coverpoint — the signal or expression being sampled
bins — partitions the coverpoint's value range into named buckets
@(posedge clk) — auto-samples at every rising edge
// LESSON 02
BINS
Types of Bins
Bins control how values are grouped. Illegal and ignore bins shape what gets counted toward coverage.
SV
covergroup opcode_cg;
  coverpoint opcode {
    // explicit — each value = 1 bin
    bins read = {4'hA};
    bins write = {4'hB};

    // wildcard bin
    wildcard bins ctrl = {4'b11??};

    // transition bin: A then B
    bins rd2wr = (4'hA => 4'hB);

    // exclude from coverage
    ignore_bins rsvd = {4'hF};

    // mark as error if hit
    illegal_bins bad = {4'h0};
  }
endgroup
bins — counts when this value/range is sampled
wildcard bins — treats ? as don't-care
transition bin (=>) — hit only when value sequence occurs in order
ignore_bins — excluded from metric, not an error
illegal_bins — triggers a simulation error if hit
// LESSON 03
CROSS
Cross Coverage
cross measures combinations of two or more coverpoints — catches scenarios missed by individual coverage.
SV
covergroup bus_cg;
  coverpoint rw {
    bins read = {0};
    bins write = {1};
  }
  coverpoint size {
    bins byte_ = {0};
    bins hword = {1};
    bins word_ = {2};
  }
  // 2 x 3 = 6 cross bins auto-created
  cross rw, size;
endgroup
cross — creates N×M bins from two coverpoints
Without cross, 100% on each coverpoint won't tell you if "write + word" was tested together
Use ignore_bins inside cross to exclude impossible combinations
// LESSON 04
OPTIONS
covergroup Options
Fine-tune sampling behaviour, goals, and bin creation using the option namespace.
SV
covergroup cfg_cg;
  // coverage goal (default 100%)
  option.goal = 90;

  // per-instance naming
  option.name = "bus_cg_inst";

  // auto-bin count limit
  option.auto_bin_max = 8;

  // bin needs N hits to be "covered"
  option.at_least = 10;

  // per-instance vs type-level
  type_option.merge_instances = 1;

  coverpoint data;
endgroup
option.goal — what % counts as "passing" for this group
auto_bin_max — tool creates up to N bins automatically
option.at_least — bin only "covered" after N hits
type_option.merge_instances — merges coverage from all instances
// LESSON 05
PARAMETERIZED
Parameterized covergroup
Pass arguments to a covergroup at instantiation for reusable, configurable coverage.
SV
covergroup mem_cg (int lo, hi);
  coverpoint addr {
    bins range = {[lo:hi]};
  }
endgroup

// Different instances for SRAM vs ROM
mem_cg sram_cg = new(0, 4095);
mem_cg rom_cg = new(4096, 8191);

// With ref argument (live sampling)
covergroup rw_cg(ref logic rw_sig);
  coverpoint rw_sig;
endgroup
rw_cg my_cg = new(dut.rw);
Arguments (lo, hi) — passed at new(), not by name. Values used in bins.
ref argument — passes a live reference; covergroup auto-samples on clock events
Great for reusing the same covergroup across multiple instances of a module
// LESSON 06
CONDITIONAL
Conditional & Guarded Coverage
Use iff to guard sampling — only sample when a condition is true, avoiding spurious hits.
SV
covergroup guarded_cg @(posedge clk);
  // only sample when valid is high
  coverpoint data iff (valid) {
    bins zero = {0};
    bins max = {8'hFF};
    bins mid = {[1:254]};
  }

  // only in non-reset state
  coverpoint state iff (!rst);
endgroup

// Dynamic enable/disable
my_cg.stop(); // pause sampling
my_cg.start(); // resume sampling
iff (condition) — per-coverpoint guard; sampling skipped when false
Prevents coverage inflation from "don't care" states like reset
stop()/start() — runtime control of the entire covergroup
// MODULE 03 — ASSERTIONS (SVA)

SystemVerilog
Assertions (SVA)

SVA lets you formally describe temporal behaviour of your design and automatically check it every clock cycle.

⚡ Immediate Assertions
✅Procedural — evaluated at a single point in time
✅No clock required — runs inside always/initial blocks
✅Simple syntax, great for X-checks
⚠️Cannot check multi-cycle behaviour
⚠️Not usable in formal verification flow
🌊 Concurrent Assertions
✅Clock-aware — checks properties across multiple cycles
✅Works in both simulation and formal verification
✅Can be placed in modules, interfaces, or programs
✅Supports disable iff for reset handling
⚠️More complex syntax to learn
// TYPE 01
IMMEDIATE
Immediate Assertions
Procedural — checked instantly at the moment of execution. Like an if-statement with error reporting.
SV
always @(posedge clk) begin
  // basic immediate assert
  assert (req !== 1'bx)
    else $error("req is X!");

  // with pass block + fail block
  assert (valid && data != 0) begin
    pass_count++;
  end else begin
    $error("data invalid!");
  end

  // deferred immediate (in final region)
  assert #0 (out == expected)
    else $error("mismatch!");

  // assume in formal
  assume (mode inside {0,1,2});
end
assert — evaluates NOW. Fails immediately if false
assert #0 — deferred: evaluates after NBA region, avoids race conditions
assume — in simulation: same as assert. In formal: constrains inputs
// TYPE 02
CONCURRENT
Concurrent Assertions
Clock-aware — checks a temporal property across multiple clock edges. The real power of SVA.
SV
// Sequence: req HIGH, then ack 1-3 cycles
sequence req_ack_seq;
  @(posedge clk)
  req ##1 ack[*1:3];
endsequence

// Property: whenever req, ack follows
property req_ack_p;
  @(posedge clk)
  disable iff (rst)
  req |=> ack;
endproperty

// Named binding for easy debug
ASSERT_REQ_ACK: assert property(req_ack_p)
  else $error("ACK missing after REQ!");

// cover — count how often it happens
COV_REQ_ACK: cover property(req_ack_p);
sequence — temporal pattern using ##, [*], etc.
property — wraps sequence with clock + disable + implication
assert property — checks every clock cycle, flags failure
disable iff — vacuously passes when condition is true (e.g. reset)

// MODULE 04 — TEMPORAL OPERATORS

Sequence Operators

The building blocks of all concurrent assertions. Master these and you can describe any protocol behaviour.

OPERATOR SYNTAX MEANING EXAMPLE
##N a ##2 b b must be true exactly N cycles after a req ##1 ack — ack follows req by exactly 1 cycle
##[m:n] a ##[1:4] b b occurs between m and n cycles after a start ##[2:5] done
[*N] a[*3] a is true for exactly N consecutive cycles valid[*4] — valid high for exactly 4 cycles
[*m:n] a[*1:4] a is true for m to n consecutive cycles hold[*2:8] — hold for 2 to 8 cycles
[+] a[+] a is true for 1 or more cycles (= [*1:$]) busy[+] — busy for at least 1 cycle
[*] a[*] a is true for 0 or more cycles (= [*0:$]) idle[*] — idle for zero or more cycles
[->N] a[->2] Non-consecutive — a true exactly N times, last occurrence ends match req[->3] — req asserted 3 non-consecutive times
[=N] a[=2] Non-consecutive — a true N times, with anything after last occurrence grant[=2] — grant seen exactly twice
|-> a |-> b Overlapping implication — check b in SAME cycle antecedent ends valid |-> !err
|=> a |=> b Non-overlapping implication — check b in NEXT cycle req |=> ack
throughout a throughout seq a must remain true during the entire sequence en throughout (req ##1 ack)
within s1 within s2 s1 must occur within the timeframe of s2 Containment checking
intersect s1 intersect s2 Both sequences start and end at same time Parallel protocol checking
first_match first_match(seq) Matches on first possible completion of sequence Useful with variable-length repeats
and s1 and s2 Both sequences must match (may end at different times) Checking parallel requirements
or s1 or s2 At least one sequence must match Alternative protocol flows
// MODULE 05 — KEYWORD REFERENCE

All SVA & Coverage
Keywords Explained

assert
Check a condition — fail if false
Used both as immediate (assert(expr)) and concurrent (assert property(p)). Simulation reports error on failure. Named assertions help with debugging.
assume
Constrain inputs for formal verification
In simulation: same as assert. In formal: tells the tool to only consider inputs satisfying this. Used to restrict state space to legal scenarios.
cover
Measure how often a property/sequence is hit
cover property(p) records every time property is true. Unlike assert, missing it doesn't error — shows as 0% coverage.
property
Named, reusable temporal specification
Wraps a sequence with a clock, optional disable iff, and logical operators. Properties can be composed. Used with assert/assume/cover.
sequence
Temporal pattern of signal behaviour
Describes multi-cycle event pattern using ##, [*], |=>, etc. Sequences can be instantiated inside properties. They're the building blocks of SVA.
disable iff
Vacuous pass condition for reset/enable
When the condition is true (e.g. reset), the property is disabled — it neither passes nor fails. Critical for avoiding false failures during reset sequences.
covergroup
User-defined coverage container type
Defined like a class, instantiated with new(). Can be triggered by clock event or sampled manually. Groups related coverpoints.
coverpoint
Signal or expression to be measured
Each coverpoint creates bins automatically or explicitly. Must be inside a covergroup. Coverage = (bins hit / total bins) × 100%.
bins
Named value bucket inside a coverpoint
A coverpoint is "covered" when all its bins are hit. Define value bins {0,1,2}, range bins [0:255], or transition bins (A => B).
ignore_bins
Exclude values from coverage metric
Values in ignore_bins don't count for or against coverage. Use for reserved/unused encodings that shouldn't affect your metric.
illegal_bins
Report error if this value is sampled
If a value matching illegal_bins is sampled, the simulator raises an error. Use to catch protocol violations like reserved opcode usage.
cross
Cross-product of two or more coverpoints
Creates N×M bins for every combination. Ensures all combinations are tested — catches corner cases that per-signal coverage misses entirely.

Advanced SVA System Functions

KEYWORD CATEGORY DESCRIPTION & EXAMPLE
$rose(sig) Sampled Value Fn True when sig transitions 0→1. $rose(req) |=> ack
$fell(sig) Sampled Value Fn True when sig transitions 1→0. Useful for detecting deassertion.
$stable(sig) Sampled Value Fn True when sig did NOT change from previous cycle.
$past(sig, N) Sampled Value Fn Returns value of sig N cycles ago. Default N=1. $past(data,2) == data
$onehot(bus) System Function True when exactly one bit is high. Great for grant bus checks: assert($onehot(grant))
$onehot0(bus) System Function True when at most one bit is high (all zeros also passes).
$countones(bus) System Function Returns number of 1-bits in bus. Use in constraints: $countones(sel) == 1
$isunknown(expr) System Function True if any bit is X or Z. Good for data bus X-checks.
not property(p) Property Operator Inverts a property. assert property(not (a ##1 b)) — pattern must NOT happen.
s_eventually LTL Operator Strong eventually — the property MUST occur at some future point. Used in formal.
s_always LTL Operator Strong always — property must hold for all time. Used in formal verification.
bind Instantiation Attaches assertions to a module without modifying source. bind dut my_sva inst(.*)
restrict Formal Like assume but only in formal — restricts the state space to valid states.
// MODULE 06 — COMPLETE PROTOCOL EXAMPLES

Real-World Protocol Examples

01
AXI4 Write Channel Assertions
AXI spec: once VALID is asserted, it must stay high until handshake (VALID & READY). READY must come within a maximum number of cycles.
AXI EXAMPLE
// Rule 1: AWVALID must not deassert before AWREADY
property awvalid_stable;
  @(posedge clk) disable iff(rst)
  (AWVALID && !AWREADY) |-> ##1 AWVALID;
endproperty
AXI_AWVALID_STABLE: assert property(awvalid_stable);

// Rule 2: AWREADY must come within 16 cycles
property awready_timeout;
  @(posedge clk) disable iff(rst)
  $rose(AWVALID) |=> ##[0:15] (AWVALID && AWREADY);
endproperty
AXI_AWREADY_TIMEOUT: assert property(awready_timeout);

// Rule 3: WDATA stable while WVALID asserted
property wdata_stable;
  @(posedge clk) disable iff(rst)
  (WVALID && !WREADY) |-> ##1 $stable(WDATA);
endproperty
AXI_WDATA_STABLE: assert property(wdata_stable);

// Cover: successful write handshake
AXI_WRITE_DONE: cover property(
  @(posedge clk) AWVALID && AWREADY && WVALID && WREADY
);
02
FIFO Protocol Assertions
A FIFO must not be written when full, not read when empty, and depth must track writes minus reads correctly.
FIFO EXAMPLE
// No write when full
property no_overflow;
  @(posedge clk) disable iff(rst)
  !(wr_en && full);
endproperty
FIFO_NO_OVERFLOW: assert property(no_overflow)
  else $fatal(1, "FIFO overflow detected!");

// No read when empty
property no_underflow;
  @(posedge clk) disable iff(rst)
  !(rd_en && empty);
endproperty
FIFO_NO_UNDERFLOW: assert property(no_underflow);

// After write, not_empty must be true
property write_makes_nonempty;
  @(posedge clk) disable iff(rst)
  (wr_en && !rd_en) |=> !empty;
endproperty
FIFO_WRITE_NONEMPTY: assert property(write_makes_nonempty);

// Full stays stable if no read/write
property full_stable;
  @(posedge clk)
  (full && !rd_en && !wr_en) |-> ##1 full;
endproperty
FIFO_FULL_STABLE: assert property(full_stable);
03
AXI Transaction Functional Coverage
Capture all meaningful combinations of AXI transaction attributes — burst type, size, length — to ensure comprehensive protocol coverage.
FUNCTIONAL COVERAGE
covergroup axi_cg @(posedge clk iff (AWVALID && AWREADY));

  // Burst type: FIXED, INCR, WRAP
  cp_burst: coverpoint AWBURST {
    bins fixed = {2'b00};
    bins incr = {2'b01};
    bins wrap = {2'b10};
    illegal_bins rsvd = {2'b11};
  }

  // Burst size: 1,2,4,8,16,32,64,128 bytes
  cp_size: coverpoint AWSIZE {
    bins byte1 = {3'd0};
    bins byte2 = {3'd1};
    bins byte4 = {3'd2};
    bins byte8 = {3'd3};
    bins byte16 = {3'd4};
  }

  // Burst length buckets
  cp_len: coverpoint AWLEN {
    bins len1 = {8'd0};
    bins len4 = {8'd3};
    bins len8 = {8'd7};
    bins len16 = {8'd15};
    bins long = {[16:255]};
  }

  // Cross: all burst_type x size combos (15 combos)
  cross cp_burst, cp_size, cp_len;

endgroup
04
SPI Bus Assertions
SPI protocol requires CS to be asserted before SCLK, data to be stable at sample edge, and CS to deassert after transfer completes.
SPI EXAMPLE
// CS must be low before first SCLK edge
property cs_before_sclk;
  @(posedge sys_clk)
  $rose(sclk) |-> !cs_n;
endproperty
SPI_CS_BEFORE_SCLK: assert property(cs_before_sclk);

// MOSI must be stable during SCLK high (sample at posedge)
sequence mosi_stable_seq;
  $rose(sclk) ##[0:1] $stable(mosi);
endsequence

// After 8 SCLK cycles, CS should deassert
property transfer_complete;
  @(posedge sys_clk)
  $rose(cs_n) |-> $past(sclk_count) == 8;
endproperty
SPI_FULL_BYTE: assert property(transfer_complete);

// Cover: back-to-back transfers
SPI_BACK2BACK: cover property(
  @(posedge sys_clk) $rose(cs_n) ##[1:3] $fell(cs_n)
);
05
Reset Sequencing & Power-Up
Ensure reset is held for minimum duration, that outputs are de-asserted during reset, and that the FSM reaches IDLE state after reset release.
RESET EXAMPLE
// Reset held for at least 4 cycles
property rst_min_width;
  @(posedge clk)
  $rose(rst) |=> rst[*4];
endproperty
RST_MIN_WIDTH: assert property(rst_min_width);

// During reset, valid_out must be 0
property out_inactive_in_rst;
  @(posedge clk)
  rst |-> !valid_out;
endproperty
RST_OUT_INACTIVE: assert property(out_inactive_in_rst);

// After reset deasserts, FSM must be IDLE within 2 cycles
property fsm_idle_after_rst;
  @(posedge clk)
  $fell(rst) |=> ##[1:2] (state == IDLE);
endproperty
FSM_IDLE_AFTER_RST: assert property(fsm_idle_after_rst);

// No X-values on output bus after reset
property no_x_after_rst;
  @(posedge clk)
  !rst |-> !$isunknown(data_out);
endproperty
NO_X_DATA: assert property(no_x_after_rst);
// INTERVIEW TIP 🎯
At AMD/Qualcomm/Intel interviews, they'll ask: "What's the difference between |-> and |=>". Answer: |-> is overlapping — consequent checked in the SAME cycle the antecedent ends. |=> is non-overlapping — consequent checked in the NEXT cycle. a |=> b is equivalent to a ##1 b |-> b.

// ADVANCED 01
UVM
Coverage in UVM Monitor
Embed covergroups inside UVM monitors. Call sample() from the write() task to capture every transaction.
UVM
class bus_monitor extends uvm_monitor;
  // Define covergroup inside class
  covergroup bus_cg;
    cp_op: coverpoint trans.op {
      bins rd = {READ};
      bins wr = {WRITE};
    }
    cp_sz: coverpoint trans.size;
    cross cp_op, cp_sz;
  endgroup

  bus_transaction trans;

  function new(...);
    super.new(...);
    // Instantiate with ref to trans
    bus_cg = new();
  endfunction

  virtual task run_phase(...);
    forever begin
      collect_transaction(trans);
      bus_cg.sample(); // sample!
    end
  endtask
endclass
Define covergroup inside the monitor class for clean encapsulation
Call sample() after every transaction is collected in run_phase
Use the transaction object's fields directly in coverpoints
// ADVANCED 02
FORMAL
Formal Verification with SVA
Use assume to constrain inputs, assert to check properties, and cover to prove reachability.
FORMAL
module arb_formal (input clk, rst, req0, req1,
                   output gnt0, gnt1);

// Assumptions: constrain legal inputs
A_NO_SIMULTANEOUS_REQ: assume property(
  @(posedge clk) !(req0 && req1)
);

// Assertions: prove these always hold
P_MUTUAL_EXCLUSION: assert property(
  @(posedge clk) disable iff(rst)
  !(gnt0 && gnt1) // never both grant
);

P_GRANT_IMPLIES_REQ: assert property(
  @(posedge clk)
  gnt0 |-> req0
);

// Reachability: prove grant is achievable
C_GNT0_REACHABLE: cover property(
  @(posedge clk) gnt0
);
C_GNT1_REACHABLE: cover property(
  @(posedge clk) gnt1
);
endmodule
assume — tells formal tool: "only explore states satisfying this"
assert — formal exhaustively proves this holds for ALL reachable states
cover — formal proves the state/scenario IS reachable (not vacuously dead)
// ADVANCED 03
BIND
bind — Non-intrusive SVA
Attach assertions to a module without modifying its source. Essential for IP and third-party blocks.
BIND
// Separate SVA module for the DUT
module fifo_sva (
  input logic clk, rst,
  input logic wr_en, rd_en,
  input logic full, empty
);

  property no_ovf;
    @(posedge clk) disable iff(rst)
    !(wr_en && full);
  endproperty

  FIFO_NO_OVF: assert property(no_ovf);

endmodule

// Bind to the actual DUT — no DUT edits!
bind fifo_dut fifo_sva fifo_sva_inst (
  .clk(clk),
  .rst(rst),
  .wr_en(wr_en),
  .rd_en(rd_en),
  .full(full),
  .empty(empty)
);
bind — instantiates fifo_sva inside fifo_dut's scope without touching fifo_dut source
The SVA module sees all DUT signals as if it were instantiated inside
Industry best practice for IP-level verification and third-party blocks
// ADVANCED 04
SEQUENCE REUSE
Reusable Sequence Library
Build a library of named sequences that can be composed into larger properties for any protocol.
SEQUENCE LIB
// Generic handshake sequence
sequence handshake(valid, ready, N);
  valid && !ready ##[1:N] valid && ready;
endsequence

// Pulse: signal high for exactly N cycles
sequence pulse(sig, N);
  $rose(sig) ##0 sig[*N] ##1 !sig;
endsequence

// Use in properties
property write_handshake_p;
  @(posedge clk)
  handshake(WVALID, WREADY, 16);
endproperty

property cs_pulse_p;
  @(posedge clk)
  pulse(!cs_n, 8);
endproperty

WRITE_HS: assert property(write_handshake_p);
CS_PULSE: assert property(cs_pulse_p);
Sequence arguments — pass signals and constants as parameters
Build a reusable protocol library (handshake, pulse, toggle, etc.)
Compose sequences into larger properties for complex multi-step checks
// ADVANCED 05
FSM COVERAGE
FSM State & Transition Coverage
Track every state and every transition in your FSM — both code coverage (tool-auto) and functional (user-written).
FSM COVERAGE
// FSM states enum
typedef enum logic [2:0] {
  IDLE=0, REQ=1, WAIT=2,
  ACK=3, DONE=4
} state_e;

covergroup fsm_cg @(posedge clk);
  // All states visited?
  cp_state: coverpoint state {
    bins idle = {IDLE};
    bins req = {REQ};
    bins wait_= {WAIT};
    bins ack = {ACK};
    bins done = {DONE};
  }

  // All transitions visited?
  cp_trans: coverpoint state {
    bins idle2req = (IDLE => REQ);
    bins req2wait = (REQ => WAIT);
    bins wait2ack = (WAIT => ACK);
    bins ack2done = (ACK => DONE);
    bins done2idle = (DONE => IDLE);
    bins wait2idle = (WAIT => IDLE); // timeout
  }
endgroup
State coverage — ensures every FSM state is visited at least once
Transition coverage — uses => syntax to track state-to-state transitions
Tools also provide automatic FSM coverage — verify both complement each other

    cp_op: coverpoint t.opcode;

      bins lo = {[0:255]};
      bins hi = {[256:511]};
    }

  endgroup

    bus_cg.sample();
  endfunction
endclass

THROUGHOUT
throughout Operator

// CS must stay low during entire transfer
property cs_throughout;
  @(posedge clk)
  $fell(cs_n) |=>
  !cs_n throughout
  (sclk[*8] ##1 $rose(cs_n));
endproperty
CS_THROUGHOUT: assert property(cs_throughout);

$PAST
$past() — History Checking
Reference signal values from previous cycles inside assertions.
SV
// Output must equal input from 2 cycles ago
property pipeline_delay;
  @(posedge clk)
  valid_out |->
  (data_out == $past(data_in, 2));
endproperty
PIPE_DELAY: assert property(pipeline_delay);

// $past with gating expression
$past(data, 1, valid_in, @clk)

endproperty

property fair_arb;

         within ##[1:4] 1'b1;

Multi-clock Domains
Handle CDC scenarios with explicit clocking blocks in sequences.
SV
// Different clocks for TX/RX
property cdc_handshake;
  @(posedge tx_clk) req |=>
  @(posedge rx_clk) ack[->1]
  within ##[1:16] 1;
endproperty

sequence fast_req;

// EDGE 06
PROTOCOL
I2C Start/Stop Detection
Detect protocol-level events using SVA sequences on I2C bus.
SV
// I2C START: SDA falls while SCL high
sequence i2c_start;
  @(posedge sys_clk)
  scl && $fell(sda);
endsequence

// I2C STOP: SDA rises while SCL high
sequence i2c_stop;
  @(posedge sys_clk)
  scl && $rose(sda);
endsequence

// Start must precede stop
property start_before_stop;
  i2c_stop |->
  i2c_start within ##[1:512] i2c_stop;
endproperty

⚡ Temporal Operators Quick Reference
a ##N b

b exactly N cycles after a

a ##[m:n] b

b between m and n cycles after a

a[*N]

a true for exactly N cycles

a[*m:n]

a true for m to n cycles

a[+]

a true for 1 or more cycles

a[*]

a true for 0 or more cycles

a[->N]

a true exactly N non-consecutive times (end on last)

a[=N]

a true N non-consecutive times (with extra after)

a |-> b

Overlapping: if a, then b in same cycle

a |=> b

Non-overlapping: if a, then b next cycle

a throughout s

a must hold during all of sequence s

s1 within s2

s1 occurs during the timeframe of s2
